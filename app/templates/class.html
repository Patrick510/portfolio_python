FIRST CONTACT WITH FLASK - EXERCISES FOR PRACTICE

All web programming works with Request and Response.
The backend uses reactive programming, meaning it doesn't start with ongoing processes, connecting the Client and Server.
Flask allows our Python code to wait for calls.

Search:
There is a special type of variable (the most common) where the name of the variable and its value are the same, like:
- __name__
- __root__

Running locally:
We turn our machines into servers.
Example:
if __name__ == '__main__':
app.run(debug=True)
- This snippet makes the code run continuously and wait for instructions, functioning as a server.
- debug=True: Allows you to edit the code with automatic updates.
- port=3030: Configures a specific port.
- host='0.0.0.0': Allows everyone on the same network to access your code.

Annotations:
- They start with @.
- Example: @app.route defines the desired route, with ('/') being the root of the project.
